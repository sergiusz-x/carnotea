# ADR-0007: i18n with Polish and English from the first screen

- **Status**: accepted
- **Date**: 2026-06-13
- **Deciders**: Sergiusz, Claude

## Context

The primary user speaks Polish; the codebase and developer-facing material is
in English. We could ship the UI in one language now and add the other later,
but retrofitting i18n into a mature app means touching every JSX file twice
(once to extract strings, once to translate).

## Decision

The web app is **bilingual from the first screen**. Every user-facing string
goes through an i18n abstraction (likely `i18next` + `react-i18next`).
Translation files exist for **Polish (`pl`)** and **English (`en`)**.

- Default language is detected from the browser, then persisted per user.
- Polish is the primary translation; English is the fallback.
- A string that exists only in one language is a build-time warning.
- Date and currency formatting uses `Intl.*` and the user's chosen locale.

## Consequences

### Positive

- No retrofit cost later. The app cannot accidentally hard-code Polish strings
  because the abstraction exists from the first commit.
- The codebase remains English (variable names, comments, error messages), which
  is the lingua franca for AI agents and external libraries.

### Negative

- Every new piece of UI requires two translation entries. We tolerate this; it
  catches developers thinking about copy as part of the work, not after it.
- Initial setup adds one library and one config step.

### Neutral

- We can add more languages later without ADR change; the architecture is
  language-count agnostic.

## Alternatives considered

### Option A: Ship Polish-only now, add English later

Rejected. Retrofit cost is high; testing matrix doubles late in the project.

### Option B: Ship English-only now, add Polish later

Rejected. The primary user is Polish. Day-one polish (pun intended) matters.

### Option C: A different i18n library (Lingui, FormatJS, etc.)

Open. The exact library is finalised in T-010. The decision in this ADR is
that *some* i18n abstraction exists from day one.
